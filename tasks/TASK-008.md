# TASK-008 — Ops Panel Frontend (внутренняя панель команды)

**Приоритет:** HIGH  
**Сложность:** Средняя  
**Зависимости:** TASK-006 (API должны работать), TASK-007 (можно переиспользовать компоненты)

---

## Цель

Создать внутренний инструмент для вашей команды. Через эту панель: создавать клиентов, настраивать Авито, смотреть логи, управлять контентом любого клиента.

---

## Контекст

Ops Panel — отдельная часть того же React-приложения. Доступна только пользователям с `role = OPS`.

Разграничение в App.tsx:
- Если `auth.role === 'OPS'` → показывать Ops Panel
- Если `auth.role === 'CLIENT'` → показывать Client Panel

Ops Panel находится по URL `/ops/...` (или hash `#/ops/...`).

─────────────────────────────────────────────────

После создания — редирект на страницу клиента с сообщением "Клиент создан. Настройте Авито-подключение."

### ClientDetail.tsx

Страница с вкладками:

**Вкладка "Обзор":**
- Статус бота (как в клиентской панели)
- Счётчики сообщений
- Кнопки: [Перезапустить бота] [Остановить] [Запустить]
- Последние 5 ошибок (если есть)
- Список объектов (только просмотр + кнопка добавить)

**Вкладка "Авито":**

```
Подключение Авито
─────────────────

Client ID:      [________________________]
Client Secret:  [________________________]  * будет скрыт после сохранения
User ID:        [________________________]
Refresh Token:  [________________________]

[Сохранить] [Тест подключения]
```

Кнопка "Тест подключения" — вызывает `POST /api/ops/clients/:id/avito/test`.

Результат теста:
- ✅ "Подключение работает. Найдено 5 чатов."
- ❌ "Ошибка: invalid_token. Проверьте Client ID и Refresh Token."

После сохранения — показывать только последние 4 символа Client Secret (остальное `****`).

**Вкладка "Контент":**

Sub-вкладки: "FAQ" | "Промпт" | "Объекты"

**FAQ** — такой же CRUD как в Client Panel, но может управлять всеми FAQ клиента.

**Промпт** — редактирование системного промпта:

```
Системный промпт бота
─────────────────────
⚠ Изменение промпта влияет на все ответы бота. Сохраняйте осторожно.

[Большой textarea с промптом]

[Сбросить к дефолтному] [Сохранить промпт]
```

При "Сбросить к дефолтному" — confirm браузера "Вы уверены? Кастомный промпт будет удалён."

**Объекты** — таблица с возможностью добавить/изменить (включая avitoItemId):

```
Название          Адрес              Avito ID      Активен
──────────────────────────────────────────────────────────

Форма добавления объекта — как в Client Panel, но с полем Avito Item ID.

**Вкладка "Логи":**

```

## Важные детали

**Отличие от Client Panel:**
- OPS может видеть и редактировать данные любого клиента
- OPS может редактировать системный промпт
- OPS может добавлять Авито-конфиг (клиент не может)
- OPS может добавлять объекты с avitoItemId (клиент не может)

**Безопасность в UI:**
- Client Secret показывать только при создании. После сохранения — маскировать `****xxxx`
- Refresh Token — показывать маскированным `****xxxx` после сохранения

## Критерии приёмки

- [ ] OPS видит список всех клиентов со статусами
- [ ] Создание клиента: форма + после создания клиент может залогиниться в Client Panel
- [ ] Страница клиента: все вкладки открываются и показывают данные
- [ ] Авито-конфиг: сохранить → тест подключения → показывает результат
- [ ] Промпт: изменить → бот использует новый промпт при следующем сообщении
- [ ] Логи: видны диалоги, история сообщений открывается
- [ ] OPS не может увидеть Ops Panel если залогинился как CLIENT (`role` проверяется)
- [ ] `npm run build` завершается без ошибок
